1) Клонирование в Git — это копирование удалённого репозитория на локальный компьютер. При этом переносятся все файлы и папки проекта, а также вся его история с момента его создания.
2) Репозиторий Git — это место, в котором хранится весь код и вся история его изменений. По сути это просто папка, однако она связана с Git напрямую и содержит файлы в понятном для Git формате.
3) Запушить в Git — это отправить все зафиксированные изменения с локального репозитория в удалённый.
